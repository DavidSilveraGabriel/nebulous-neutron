## 1. `try`/`except`/`finally` <Badge text="Capturando Errores" variant="tip" />

La estructura `try`/`except`/`finally` se utiliza para capturar y manejar excepciones que pueden ocurrir durante la ejecuci√≥n de un bloque de c√≥digo.

### Estructura `try`/`except`

El bloque `try` contiene el c√≥digo que puede generar una excepci√≥n. Si se produce una excepci√≥n, el bloque `except` correspondiente la captura y la maneja.

```python
try:
    resultado = 10 / 0  # Esto genera una ZeroDivisionError
except ZeroDivisionError:
    print("¬°No se puede dividir por cero!")
```

### Estructura `try`/`except` con Tipos de Excepciones Espec√≠ficos
Se pueden especificar distintos bloques `except` para distintos tipos de excepciones.

```python
try:
  numero = int("Hola") # genera un ValueError
  resultado = 10 / numero # puede generar un ZeroDivisionError
except ValueError:
  print("El valor introducido no es un n√∫mero")
except ZeroDivisionError:
  print("No se puede dividir por cero")
```

### Estructura `try`/`except` con Bloque `else`
Se puede incluir un bloque `else` que se ejecuta cuando el bloque `try` no produce ninguna excepci√≥n.

```python
try:
  resultado = 10/2
except ZeroDivisionError:
  print("No se puede dividir por cero")
else:
  print(f"El resultado es {resultado}") # El resultado es 5.0
```

### Estructura `try`/`except`/`finally`

El bloque `finally` se ejecuta siempre, tanto si se produce una excepci√≥n como si no, siendo muy √∫til para realizar tareas de limpieza y cierre de recursos.

```python
archivo = None

    archivo = open("mi_archivo.txt", "r")
    contenido = archivo.read()
    print(contenido)
except FileNotFoundError:
    print("El archivo no se encuentra")
finally:
    if archivo:
       archivo.close()
```
### Capturar m√∫ltiples excepciones en un solo bloque `except`

```python
try:
    archivo = open("mi_archivo.txt", "r")
    contenido = archivo.read()
    print(contenido)

    print(f"Error al abrir el archivo: {error}")
```

### Capturar cualquier excepci√≥n

Se puede capturar cualquier tipo de excepci√≥n usando un `except` sin especificar el tipo de error.

```python

    numero = 10/0
except Exception as e:
    print(f"Ha ocurrido un error: {e}") # Ha ocurrido un error: division by zero
```
<Aside type="note" title="Nota üìù">
   Es mejor capturar excepciones espec√≠ficas en lugar de usar un bloque `except` gen√©rico, para poder manejar cada tipo de error de forma adecuada.
</Aside>

## 2. Creaci√≥n de Excepciones <Badge text="Tus Propias Excepciones" variant="note" />

Python permite crear clases personalizadas de excepciones para manejar situaciones espec√≠ficas de tu aplicaci√≥n.

### Pasos para Crear una Excepci√≥n Personalizada
1.  **Crea una clase que herede de `Exception` o una de sus subclases:** (ej: `ValueError`).
2.  **Define un constructor `__init__` que reciba un mensaje de error (opcional):**

    ```python
    class SaldoInsuficienteError(Exception):
        def __init__(self, mensaje):
            self.mensaje = mensaje
            super().__init__(self.mensaje)

    def retirar_dinero(saldo, cantidad):
      if saldo < cantidad:
        raise SaldoInsuficienteError("Saldo insuficiente")
      else:
         return saldo - cantidad

    try:
        saldo_restante = retirar_dinero(100,200)
        print(saldo_restante)
    except SaldoInsuficienteError as error:
        print(f"Error al retirar dinero: {error}") # Error al retirar dinero: Saldo insuficiente
    ```

### Ventajas de las Excepciones Personalizadas
*   **C√≥digo m√°s claro:** Facilita la identificaci√≥n de errores espec√≠ficos de tu aplicaci√≥n.
*   **Manejo de errores m√°s preciso:** Permite crear l√≥gica espec√≠fica para cada tipo de excepci√≥n.
*   **Mensajes de error m√°s informativos:** Facilita la depuraci√≥n.

## 3. Logging y Debugging <Badge text="Rastreando Errores" variant="success" size="large" />

El logging y debugging son herramientas esenciales para rastrear, analizar y corregir errores en tu c√≥digo.

### Logging
El m√≥dulo `logging` proporciona un sistema flexible para registrar informaci√≥n de diagn√≥stico y errores durante la ejecuci√≥n de tu aplicaci√≥n.

*   **`logging.basicConfig(...)`:** Configura el registro b√°sico (nivel, formato, archivo, etc.).
*   **`logging.debug(mensaje)`:** Registra un mensaje de debug.
*   **`logging.info(mensaje)`:** Registra un mensaje informativo.
*   **`logging.warning(mensaje)`:** Registra un mensaje de advertencia.
*   **`logging.error(mensaje)`:** Registra un mensaje de error.
*   **`logging.critical(mensaje)`:** Registra un mensaje de error cr√≠tico.

```python
import logging

logging.basicConfig(level=logging.DEBUG, format='%(asctime)s - %(levelname)s - %(message)s')
logging.debug("Esto es un mensaje de debug")
logging.info("Esto es un mensaje informativo")
logging.warning("Esto es un mensaje de advertencia")
logging.error("Esto es un mensaje de error")
logging.critical("Esto es un mensaje de error cr√≠tico")

try:
  resultado = 10/0
except Exception as e:
    logging.error(f"Error al dividir: {e}")
```

### Debugging
Python proporciona herramientas para depurar tu c√≥digo y rastrear los errores de forma interactiva.

*   **`print()` (para debugging simple):**  √ötil para mostrar el valor de variables.

    ```python
    def sumar(a, b):
        print(f"sumar: a={a}, b={b}")
        resultado = a + b
        print(f"sumar: resultado={resultado}")
        return resultado
    sumar(5,3)
    ```
*   **`breakpoint()`:** Inserta un punto de interrupci√≥n para iniciar el debugger.
    ```python

        breakpoint()
        resultado = a + b
        return resultado
    sumar(5,3)
    ```
*   **`pdb` (Python Debugger):** Inicia el debugger de Python.
    *   Ejecutar en consola: `python -m pdb tu_script.py`.
    *   Comandos √∫tiles: `n` (next), `s` (step), `c` (continue), `b` (breakpoint), `p` (print), `q` (quit).

<Aside type="tip" title="Consejo ‚ú®">
   El logging es ideal para registrar eventos en aplicaciones en producci√≥n, mientras que el debugging es m√°s √∫til durante el desarrollo para encontrar y corregir errores.
</Aside>

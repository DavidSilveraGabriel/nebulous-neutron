## 1. Docker <Badge text="Contenedores para un Entorno Consistente" variant="tip" />

Docker es una plataforma para crear, gestionar y ejecutar aplicaciones en contenedores. Los contenedores permiten empaquetar una aplicaci√≥n junto con todas sus dependencias, asegurando que se ejecute de la misma manera en cualquier entorno.

### Instalaci√≥n de Docker
*   Descargar e instalar Docker Desktop o Docker Engine en tu sistema operativo.

### Conceptos B√°sicos de Docker

*   **Imagen:**  Es un archivo con las instrucciones para construir un contenedor, es una plantilla de un contenedor.
*   **Contenedor:** Es una instancia en ejecuci√≥n de una imagen.
*   **Dockerfile:**  Es un archivo de texto que contiene las instrucciones para construir una imagen de Docker.
*   **Docker Compose:** Es una herramienta para definir y ejecutar aplicaciones multi-contenedor.

### Ejemplo de un Dockerfile
```dockerfile
# Dockerfile
FROM python:3.9-slim

WORKDIR /app

COPY requirements.txt .
RUN pip install -r requirements.txt

COPY . .

CMD ["python", "main.py"]
```
### Construir una Imagen de Docker

```bash
docker build -t mi-imagen .
```
*  `docker build`: Construye una imagen de Docker.
*   `-t mi-imagen`: Asigna el tag `mi-imagen` a la imagen.
*   `.`: Indica que el Dockerfile se encuentra en el directorio actual.

### Ejecutar un Contenedor de Docker

```bash
docker run -p 8000:8000 mi-imagen
```
* `docker run`: Ejecuta un contenedor a partir de una imagen.
* `-p 8000:8000`: mapea el puerto 8000 del host al puerto 8000 del contenedor.
* `mi-imagen`: es el tag de la imagen a ejecutar.

### Docker Compose
* Permite configurar y gestionar aplicaciones multi-contenedor a traves de un archivo `docker-compose.yml`.
*  Se utiliza para gestionar aplicaciones que necesitan m√∫ltiples contenedores (ej: una aplicaci√≥n web con una base de datos).

```yaml

  web:
    build: .

      - "8000:8000"

      - db
```

```bash

docker-compose down  # detiene los contenedores
```

## 2. CI/CD B√°sico <Badge text="Automatizando el Desarrollo" variant="note" />

CI/CD (Continuous Integration / Continuous Deployment) es una pr√°ctica de DevOps que automatiza el proceso de integraci√≥n de c√≥digo, pruebas y despliegue de aplicaciones.

### Componentes del CI/CD
*   **Integraci√≥n Continua (CI):**  Integra los cambios de c√≥digo de forma frecuente y automatizada, ejecutando pruebas para detectar errores.
*   **Despliegue Continuo (CD):**  Despliega los cambios de c√≥digo en un entorno de producci√≥n de forma automatizada.

### Herramientas de CI/CD
*   **GitHub Actions:** Una plataforma para automatizar flujos de trabajo de CI/CD dentro de GitHub.
*   **GitLab CI:** Una plataforma para automatizar flujos de trabajo de CI/CD dentro de GitLab.
*   **Jenkins:**  Una herramienta de c√≥digo abierto para automatizar la integraci√≥n y el despliegue continuo.
* **Travis CI**: Es una herramienta de CI/CD que automatiza la creaci√≥n de software.

### Ejemplo de un Flujo de CI/CD B√°sico con GitHub Actions
*  Se define un archivo `.github/workflows/main.yml` para definir un flujo de trabajo.
* El flujo de trabajo define los pasos a seguir (ej:  `checkout` , `set-up-python`, `install`, `test` y `deploy` ).
```yaml
# .github/workflows/main.yml
name: CI/CD
on:
  push:
    branches: [ main ]
jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout code
        uses: actions/checkout@v3
      - name: Set up Python
        uses: actions/setup-python@v4
        with:
          python-version: '3.9'
      - name: Install dependencies
        run: pip install -r requirements.txt
      - name: Run tests
        run: pytest
      - name: Deploy
        run: echo "Deploying to production"

```
### Pasos del Flujo de CI/CD
1.  Los desarrolladores hacen cambios y hacen un `push` a un repositorio Git.
2. La plataforma de CI/CD (GitHub Actions, GitLab CI, etc) detecta el `push` al repositorio y ejecuta el flujo de trabajo.
3.  El flujo de trabajo instala dependencias, ejecuta pruebas, y en caso de exito despliega la aplicaci√≥n.
4.  En caso de errores el flujo de trabajo notifica a los desarrolladores.

<Aside type="caution" title="Precauci√≥n ‚ö†Ô∏è">

## 3. Deployment <Badge text="Poniendo tu Aplicaci√≥n en Producci√≥n" variant="success" size="large" />

El deployment (despliegue) es el proceso de poner tu aplicaci√≥n en funcionamiento en un entorno de producci√≥n para que los usuarios puedan acceder a ella.

### Estrategias de Deployment

*   **Despliegue con Contenedores:**  Utilizar Docker para empaquetar y desplegar la aplicaci√≥n en contenedores.

### Ejemplo de Despliegue con Docker
1. Crear un archivo Dockerfile (ej:  ver la seccion de Docker).
2. Construir una imagen Docker: `docker build -t mi-app`.
3. Subir la imagen a un registro de im√°genes como Docker Hub o GitHub Container Registry: `docker push mi-usuario/mi-app`.
4. En el servidor de producci√≥n, ejecutar el contenedor usando: `docker run -p 80:80 mi-usuario/mi-app`.
